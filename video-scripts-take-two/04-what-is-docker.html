Video 4

What Is Docker?

Make Docker feel practical: containers are the packaged services, and Compose is the recipe that starts them together.

Target: about 2 minutes Start: Docker Desktop or terminal Finish: running stack

Plain narration: 04-what-is-docker-transcript.txt

Before Recording

  • Have the Paperless Ag local stack running so Docker Desktop or docker compose ps shows real containers.
  • Show containers with friendly names: Paperless, Postgres, Redis, companion, and optionally Caddy in an install environment.
  • Avoid turning this into a Docker tutorial. The purpose is to remove fear before the install scripts use Docker.

Choreography

Time On Screen Narration
0:00-0:25 Show Docker Desktop with the Paperless stack running, or a terminal ready to run docker compose ps. Hey this is Brian from Fullstack Ag. When we install Paperless Ag, you are going to see Docker mentioned a lot. Docker sounds like a programmer thing, but for this project you can think of it as the packaging system that keeps the whole setup predictable.
0:25-0:55 Show Point at individual running containers. Paperless is not just one program. It has the Paperless web app, a Postgres database, Redis for background work, and Paperless Ag for semantic search and Claude. Docker puts each piece in its own container so the pieces do not get tangled up with the rest of the computer.
0:55-1:25 Type In terminal, run docker compose ps and show the running services. Docker Compose is the recipe that starts the containers together. Instead of installing a database by hand, then Redis by hand, then Paperless by hand, the Compose file says: here are the services, here is how they connect, and here is where the data should live.
1:25-1:55 Show Keep the running service list visible. Optionally point at volumes in Docker Desktop if available. That last part matters. Containers can be replaced, updated, or restarted, but your documents and database live in persistent storage. So Docker gives us a clean way to update the software without treating your data as disposable.
1:55-2:20 Show End on Docker Desktop or terminal with all services healthy. The reason we use Docker is that the same stack can run on my Mac, on a DigitalOcean server, or on a Raspberry Pi with the same basic commands. You do not need to become a Docker expert. The installer handles the boring parts for you. Thanks for watching!
