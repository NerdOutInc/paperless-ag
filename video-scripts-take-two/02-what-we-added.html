What We Added with Paperless Ag

Keep this one centered on the payoff: keyword search can miss the right document, while Paperless Ag lets Claude search by meaning.

Target: 3-4 minutes Start: Paperless search Finish: Claude answer

Plain narration: 02-what-we-added-transcript.txt

Before Recording

  • Local stack running with 100 demo documents and embeddings covering all documents.
  • Paperless Ag MCP connector already working in Claude Desktop. This version avoids the long config-editing sequence.
  • Paperless keyword demo: fertilizer recommendations, typed with no quotes, should return zero documents in the demo data.
  • Claude prompt: What are my fertilizer recommendations? Search my Paperless Ag documents.

Choreography

Time On Screen Narration
0:00-0:25 Show Start on Paperless NGX with the full demo archive visible. Hey this is Brian from Fullstack Ag. In the last video, we saw Paperless NGX turn farm paperwork into a searchable archive. That is already useful, but normal search still has one annoying problem.
0:25-0:55 Type Search fertilizer recommendations in Paperless. Show the empty result state. You have to guess the words the document actually used. If I search for fertilizer recommendations, Paperless comes back empty. But I know those records exist. They are just named things like Nutrient Management Plan and they mention products like anhydrous ammonia instead of the words I typed.
0:55-1:35 Show Hold on the failed search, then switch to paperless.fullstack.ag or a simple architecture section on the project page. Paperless Ag fixes that by adding semantic search. It does not replace Paperless NGX and it does not modify Paperless itself. It runs beside it as a companion container, reads the text Paperless already extracted, and builds a local search index based on meaning.
1:35-2:05 Show Show the project page bullets or keep Paperless visible while explaining the architecture. The technical version is: it splits documents into chunks, creates embeddings with a small local model, stores those embeddings in Postgres with pgvector, and exposes search tools through MCP. The plain-English version is: Claude gets a safe way to search your Paperless archive.
2:05-2:50 Type Switch to Claude Desktop with the paperless-ag connector visible. Ask the fertilizer question and submit with Return. So now I can ask the question the way I would say it out loud: what are my fertilizer recommendations? Claude calls the Paperless Ag search tool, searches the archive by meaning, and finds the nutrient plans that keyword search missed.
2:50-3:35 Wait Let Claude answer. Slowly scroll the response so the source documents and recommendation details are visible. That is the important difference. I do not have to remember the exact title, the exact product name, or the exact phrase. I can describe the thing I need, and Claude can bring back the right documents with enough context to answer.
3:35-4:00 Show Hold on Claude's answer with the connector context visible. The indexing happens on your server, using your Paperless data and the Postgres database Paperless already needs. When you ask Claude a question, it gets the search results it needs to help you. Paperless stays stock, and Paperless Ag adds the smarter layer beside it. Thanks for watching!
