# Fullstack AG Docker Slides Design

## Style Prompt

Warm technical explainer for agricultural operators. The Docker visuals should
feel like the VPS HyperFrames segment and the existing Fullstack AG title-card
assets: practical, calm, and operator-focused. Use dark field-inspired canvases,
cream work surfaces, sage infrastructure accents, burnt-orange emphasis,
numbered marks, terminal/system labels, and restrained copy. The goal is to make
Docker feel like predictable packaging, not a developer-only rabbit hole.

## Colors

- `#17120f` charcoal: dark canvas, terminal surfaces, deep UI chrome
- `#5f7358` sage: field texture, container accents, service connection lines
- `#f5f1eb` cream: primary text and light panels
- `#d4622b` burnt orange: section badges, key strokes, status emphasis
- `#3f2f27` chocolate: secondary panels and warm shadow depth

## Typography

- Manrope: headings, section numbers, large operator-facing labels
- Inter: body copy and plain-language supporting text
- JetBrains Mono: terminal commands, service labels, technical readouts

## Motion Rules

- Use slow field-line drift and light grain so the slides feel alive without
  becoming flashy.
- Use entrance animations for every explanatory scene element.
- Use warm push/blur-style transition washes between slide scenes.
- Preserve the real Docker Desktop and Terminal footage for the middle demo.
- Keep all copy readable in two seconds; this is a narrated tutorial, not an ad.

## What NOT to Do

- Do not use neon blue/purple tech gradients.
- Do not rely on generic cloud icons or Docker-logo-only visuals.
- Do not make the slides look like polished SaaS marketing.
- Do not replace the real Docker Desktop or `docker ps` demo section.
- Do not overwrite the existing finished MP4.
