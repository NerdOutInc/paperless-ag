Video 4: What Is Docker?

Target: 2-3 minutes Type: Concept (standalone)

Before You Record

  • Docker Desktop open (on your Mac)
  • Have a few containers running (the Paperless stack is perfect) so Docker Desktop isn't empty
  • Terminal open in a second window or tab
Hook — The Analogy (0:00 - 0:30)
On Screen Narration
Show Docker Desktop main screen, showing running containers. "When we set up Paperless, we're going to use something called Docker. If you've never heard of it, don't worry — it's simpler than it sounds."
Plain English Explanation (0:30 - 1:15)
On Screen Narration
Show Docker Desktop, containers list. Point is to have a visual while you explain. "Think of Docker like a shipping container — the kind you see on trains and cargo ships. The whole point of a shipping container is that it doesn't matter what's inside. Could be grain, could be tractor parts. It fits on any truck, any ship, any train. You don't have to worry about how to load it — it just works."
Show Still Docker Desktop. "Docker does the same thing for software. Instead of installing a program and hoping it works on your computer, everything the program needs gets packed into a container. You run the container and it just works — doesn't matter if you're on a Mac, a Linux server, or a NAS in your shop."
What It Looks Like (1:15 - 2:00)
On Screen Narration
Click In Docker Desktop, click on the Paperless-related containers. Show the container list — db, redis, paperless-webserver, app. Show them running (green status). "Here's what it actually looks like. These are the containers running Paperless right now on my computer. There's a few of them working together — one for the database, one for Paperless itself, and one for the search add-on we built."
Click Switch to the Terminal. Run docker compose ps to show running containers in the terminal too. "You can see them in the terminal too. Each one is running in its own little box, keeping things clean and separate."
Why Docker Matters for You (2:00 - 2:30)
On Screen Narration
Show Docker Desktop with the running containers visible. "The reason this matters for you is that you don't have to be a programmer to set this up. Docker handles all the complicated parts. You basically say 'run this' and it pulls everything down and starts it up. If something breaks, you throw away the container and start fresh. Your data is stored separately, so nothing gets lost."
Wrap-Up (2:30 - 2:45)
On Screen Narration
Show Docker Desktop, containers running. "That's Docker — it packages software so it runs the same everywhere, and you don't have to worry about installing dependencies or breaking things. In the install videos, you'll see that our setup script handles all the Docker stuff for you automatically."
