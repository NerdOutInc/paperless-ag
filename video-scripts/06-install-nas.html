Video 6: How to Install on QNAP / Synology

Target: 8-12 minutes Type: Install walkthrough

Before You Record

  • QNAP NAS accessible on your local network
  • Container Station installed on your QNAP (install from App Center if not already there)
  • Browser open to your QNAP admin interface (http://NAS-IP:8080 or similar)
  • Have the docker-compose.yml ready to paste (the production version from the repo or a NAS-adapted version with adjusted volume paths)
  • Prepare a version of docker-compose.yml with QNAP volume paths (e.g., /share/Container/paperless/...) before recording
NAS hardware note: Your TS-464 has an Intel Celeron quad-core with 8 GB RAM — plenty for the full stack including the embedding model. Synology users need an Intel-based model (DS224+, DS423+, DS923+, etc.) and DSM 7.2+ for Container Manager.
Intro (0:00 - 0:45)
On Screen Narration
Show QNAP admin web interface (QTS desktop). Show it briefly so viewers see what QTS looks like. "If you've got a NAS sitting in your house — a QNAP or a Synology — you can run Paperless right on it instead of paying for a cloud server. Your documents stay on your own hardware, on your own network. I've got a QNAP TS-464, so I'll walk through this on my actual NAS."
Show Still on the QTS desktop. "If you're on a Synology, the steps are almost identical — they call their version Container Manager instead of Container Station, but it works the same way. I'll call out the differences as we go."
Install Container Station (0:45 - 2:30)
On Screen Narration
Click Open the App Center on QNAP. Search for "Container Station." "First we need to install Container Station. This is QNAP's app for running Docker containers. Open the App Center and search for 'Container Station.'"
Click If not installed, click Install. If already installed, show that it's there and open it. "Hit Install if you don't have it yet. It's free from QNAP. On a Synology, you'd open Package Center and install Container Manager — same idea."
Click Open Container Station. Show the main interface — the overview or containers list. "Once it's installed, open it up. This is where we'll set up Paperless."
Deploy with Docker Compose (2:30 - 5:30)
Synology note: On Synology Container Manager, go to Project → Create. Paste the same compose file but use /volume1/docker/paperless/... for volume paths instead.
On Screen Narration
Click In Container Station, navigate to Applications (or Create Application). Click "Create." "Go to Applications and click Create. This is where we paste in our setup file — it's called a Docker Compose file, and it tells the NAS exactly what to download and run."
Paste Paste the docker-compose.yml contents into the YAML editor. Make sure volume paths use QNAP paths like /share/Container/paperless/data instead of Docker named volumes where needed. "I'll paste in our Compose file. This sets up everything — the database, Paperless NGX, and our search add-on. The one thing you need to adjust is the file paths. On a QNAP, your storage is under '/share/Container' — I've already updated those."
Click Set the application name to "paperless" or "paperless-ag." "Give it a name — I'll call it 'paperless' — and let's fire it up."
Click Click Create / Deploy / Apply to start the stack. "Hit Create, and it starts pulling everything down."
Wait / Cut Container Station will pull Docker images. This takes a few minutes. Show progress briefly, then fast-forward to when all containers are running (green status). "This is downloading all the pieces — Paperless, the database, our search add-on. Takes a few minutes on the first run. I'll skip ahead… And they're all running."
Verify It Works (5:30 - 7:00)
On Screen Narration
Click Open a new browser tab. Navigate to http://NAS-IP:8000. The Paperless login page should appear. "Let's check it. Open a new tab and go to your NAS's IP address, port 8000."
Type Log in with the admin credentials you set in the compose file. "Log in with the username and password from the compose file, and there's Paperless — running right on your NAS."
Click Drag and drop a test PDF into Paperless to show it working. "Let's drop in a test document to make sure everything's connected. Drag a PDF in… and there it goes, processing. Paperless reads the text, and our add-on will automatically build the search index for it."
Connect to Claude (7:00 - 9:30)
On Screen Narration
Show Paperless running in the browser. Pause here. "Now let's connect Claude. Since the NAS is on your home network, we need the local IP address and the port where our search add-on is running — that's port 3001."
Click Open Claude Desktop. Go to Settings → Developer → Edit Config. "Open Claude Desktop, go to Settings, Developer, Edit Config."
Type Paste the Claude Desktop MCP config, using http://NAS-IP:3001/mcp as the URL and your MCP token. Save the file. "Add the MCP server config. The URL is your NAS IP address, port 3001, path /mcp. And the token from your compose file's environment variables. Save it."
Click Restart Claude Desktop. Open a new chat. Verify MCP connection (tools icon). "Restart Claude, open a new chat, and check for the tools icon. There it is — Claude is connected to your NAS."
Type Ask Claude: "When was my John Deere last serviced?" It should find the equipment service records. "Let's test it — 'When was my John Deere last serviced?' And it searches right through to the NAS and finds the service record. Everything's running locally on your own hardware."
Tips & Considerations (9:30 - 10:30)
On Screen Narration
Show Container Station showing all containers running healthy. "A few things to keep in mind. Your NAS needs an Intel processor and at least 4 gigs of RAM for this. My QNAP has 8, which gives it plenty of headroom. Most newer QNAP and Synology models with Intel chips will work fine."
Show Same screen, no action needed. "Since this runs on your home network, you can only access it when you're at home — unless you set up a VPN or remote access, which is a topic for another day. If you need access from the field, the DigitalOcean setup from our other video might be a better fit."
Wrap-Up (10:30 - 11:00)
On Screen Narration
Show Paperless running in the browser with documents visible. "And that's it — Paperless with smart search running right on your NAS. No monthly fees, no cloud subscription, your documents never leave your house. Start scanning in that paperwork and let Claude help you find whatever you need."
