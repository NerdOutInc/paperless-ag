Video 6: How to Install on a Raspberry Pi

Target: 10-14 minutes Type: Install walkthrough

Before You Record

  • Raspberry Pi 4 (4 GB minimum) or Pi 5 (8 GB recommended), power supply, and an SD card or USB SSD (256 GB+ recommended)
  • Raspberry Pi Imager installed on your computer (raspberrypi.com/software)
  • Ethernet cable plugged into the Pi (recommended) or Wi-Fi credentials ready for the Imager
  • Terminal app open on your recording computer, ready to SSH into the Pi
  • Have the install command ready to paste: curl -fsSL https://paperless.fullstack.ag/install.sh | bash
  • Plan to cut / fast-forward the SD-card flash (2-5 min), first-boot wait (1-2 min), and Docker image pulls (5-15 min on a Pi)
  • Router admin page open in a tab if you plan to demo DHCP reservation
Hardware note: Pi 4 with 2 GB RAM will not work — Paperless, Postgres, Redis, and the embedding model need at least 4 GB together. A USB SSD is strongly recommended over an SD card for database workloads.
Intro (0:00 - 0:45)
On Screen Narration
Show Close-up of the Raspberry Pi hardware on your desk — Pi, power supply, SSD or SD card visible. "If you'd rather run Paperless at home instead of paying for a cloud server, a Raspberry Pi is one of the cheapest ways to do it. It's small, quiet, sips power, and handles this workload just fine. I'll walk through the whole thing — from flashing the SD card to searching documents with Claude."
Show Still on the Pi hardware. Point at the USB port where the SSD plugs in if you have one. "You'll want a Pi 4 with at least 4 gigs of RAM, or a Pi 5. If you have a USB SSD, plug it in — the database runs a lot better off a real drive than off an SD card."
Flash Raspberry Pi OS (0:45 - 3:00)
On Screen Narration
Click Open Raspberry Pi Imager on your computer. Plug the SD card (or SSD) into the computer. "First step — we need to install an operating system. Open up Raspberry Pi Imager. If you don't have it, grab it from raspberrypi.com/software — it's free."
Click Click "Choose Device" and pick your Pi model (Pi 4 or Pi 5). "Pick your Pi model up top."
Click Click "Choose OS." Navigate to Raspberry Pi OS (other) → Raspberry Pi OS Lite (64-bit). "For the OS, pick Raspberry Pi OS Lite, 64-bit. We don't need the desktop version — this is going to be a headless server, so skipping the desktop saves us a bunch of RAM."
Click Click "Choose Storage" and select your SD card or SSD. "Pick your storage — the SD card or the SSD you're flashing."
Click Click "Next," then "Edit Settings" when prompted to apply OS customization. The settings dialog opens. "Before we flash, the Imager will ask if we want to customize settings. Hit Edit Settings — this is where we pre-configure the Pi so we don't have to hook up a monitor and keyboard later."
Type In the General tab: set hostname to paperless, username + password, and Wi-Fi credentials if you're not using Ethernet. Set locale / timezone. "Set the hostname to 'paperless' so it's easy to find on the network. Pick a username and a good password. If you're on Wi-Fi, enter your network name and password. Set your timezone."
Click Switch to the Services tab. Enable SSH with password authentication. "Over on the Services tab, turn on SSH. That's how we'll log into the Pi from our computer."
Click Click Save, then Yes to apply settings, then Yes to confirm erasing the drive. Flashing begins. "Save, confirm, and the Imager flashes the card. This takes a few minutes."
Wait / Cut Flashing + verification takes 2-5 minutes. Cut to the finished state. "I'll skip ahead… And it's done."
Boot the Pi & SSH In (3:00 - 4:30)
On Screen Narration
Show The Pi hardware. Plug the SD card (or SSD) into the Pi. Plug in Ethernet, then power. "Pop the card into the Pi, plug in your network cable, and give it power. The first boot takes about a minute while the OS expands itself onto the card."
Click Switch to your Terminal on your computer. "Once it's had a minute to boot, switch over to your terminal."
Type Run ssh YOUR_USERNAME@paperless.local. Accept the fingerprint prompt, type the password you set earlier. "Type 'ssh', your username, at paperless dot local. That hostname is what we set in the Imager. If it asks about a fingerprint, type 'yes.' Then enter your password."
Fallback: If paperless.local doesn't resolve (some routers don't do mDNS), find the Pi in your router's admin page under connected devices, and SSH in by IP instead: ssh YOUR_USERNAME@192.168.1.xxx. "If 'paperless.local' doesn't work — some routers don't support that — just check your router's admin page for connected devices, find the Pi, and use its IP address instead. Either way, same result."
Show Terminal showing the Pi's shell prompt (user@paperless:~ $). "We're in — that's the Pi's command line."
Run the Install Script (4:30 - 7:30)
On Screen Narration
Type Paste: curl -fsSL https://paperless.fullstack.ag/install.sh | bash "Now the good part — one command does everything. This is the same script we used for the DigitalOcean install. It figures out we're on a Pi automatically."
Wait Installer checks system resources (RAM, disk). If Docker isn't installed it prompts to install it. "It checks that the Pi has enough memory and disk space. Then it asks if we want to install Docker — yes, we do."
Gotcha: After Docker installs, the script tells you to log out and back in (Docker needs a group change). When that happens, exit SSH, SSH back in, and re-run the same install command. "Heads up — after Docker installs, it'll tell you to log out and log back in. That's normal. Just type 'exit,' then SSH back in and re-run the same install command. It picks up where it left off."
Type Walk through the installer prompts:
- Install directory (accept default ~/paperless-ag)
- Admin username + password for Paperless
- Timezone
- Domain name (press Enter to skip — local network install) 		"It asks a handful of questions. Install directory — the default's fine. Username and password for Paperless. Timezone. Domain name — since we're on our home network, just hit enter to skip."
Wait / Cut Docker pulls about 2 GB of images on ARM. On a Pi this takes 5-15 minutes depending on internet speed. Cut to the finished state. "Now it's pulling down all the images — Paperless, Postgres, Redis, and our search add-on. On a Pi this is the slow part. I'll skip ahead…"
Show The installer's final summary: Paperless URL, MCP URL, MCP auth token. Pause so viewers can see it. "Done. And the script prints out everything we need — the URL to access Paperless, the MCP endpoint for Claude, and the auth token. Copy that token somewhere safe, we'll need it in a minute."
Verify on the Network (7:30 - 9:00)
On Screen Narration
Click Open a browser tab on your computer. Navigate to http://paperless.local (or http://PI-IP). "Open a browser and go to paperless dot local — or the Pi's IP address. The installer sets up a reverse proxy, so it's just plain port 80, no extra port number needed."
Type Log in with the admin username and password you chose during install. "Log in with the username and password you just set. And there it is — Paperless, running on the Pi."
Click Drag and drop a PDF into Paperless to show it ingesting. The document appears in the list after a moment. "Drop in a test PDF to make sure everything's wired up. Paperless picks it up, OCRs it, and our add-on builds the search index in the background."
Show Briefly mention DHCP reservation. Optionally flip to the router admin page showing the DHCP leases. "Quick tip — go into your router's admin page and give the Pi a DHCP reservation. That locks it to the same IP forever, so bookmarks don't break the next time it reboots."
Connect to Claude (9:00 - 11:00)
On Screen Narration
Show Switch back to the terminal where the install summary is still visible with the MCP URL and auth token. "Last step — connect Claude so we can search with plain English. Grab the MCP URL and auth token from the install output."
Click Open Claude Desktop. Go to Settings → Developer → Edit Config. "Open Claude Desktop, Settings, Developer, Edit Config."
Type Paste the MCP config the installer generated. URL is http://paperless.local/mcp (or the Pi's IP) with the auth token in the Authorization header. Save the file. "Paste in the config from the install output. The URL is your Pi's address plus slash mcp, and the auth token goes in the header. Save it."
Click Quit Claude Desktop completely and reopen it. Start a new chat. Verify the MCP tools indicator shows paperless-ag connected. "Quit Claude and reopen it so it picks up the new config. Start a new chat, and you should see paperless-ag in the tools list — that means Claude is talking to the Pi."
Type Ask Claude a test query, e.g., "When was my John Deere last serviced?" Watch it call the search_documents tool and pull back the equipment service record. "Let's try it — 'When was my John Deere last serviced?' And there it goes, searches the Pi, pulls back the equipment service record. Everything running locally on hardware you own."
Tips & Considerations (11:00 - 12:00)
On Screen Narration
Show Pi hardware sitting on the desk with the status LED blinking. "A few things worth knowing. Use a USB SSD if you can — an SD card works, but the database is faster and more reliable off a real drive. Ethernet beats Wi-Fi if you have the option. And the first big batch of documents will take a while to embed on ARM, but it runs in the background, so you can keep using Paperless while it catches up."
Show Same shot. "Since this is on your home network, you can only get to it from home unless you set up a VPN or remote access — that's a topic for another day. If you need access from anywhere, the DigitalOcean setup from our other video is the better fit."
Wrap-Up (12:00 - 12:30)
On Screen Narration
Show Paperless running in the browser, Claude response visible in the other window. "And that's it — Paperless with smart search running on a Raspberry Pi in your house. No monthly fees, no cloud subscription, your documents never leave the building. Plug in a scanner and start feeding it paperwork."
