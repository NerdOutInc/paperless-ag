Video 5: How to Install on DigitalOcean

Target: 8-12 minutes Type: Install walkthrough

Before You Record

  • DigitalOcean account logged in
  • SSH key already added to your DO account (or be ready to create one during recording)
  • Terminal app open (iTerm, Terminal, etc.)
  • Have the install command ready to paste: curl -fsSL https://paperless.fullstack.ag/install.sh | bash
  • Plan to fast-forward / cut the droplet creation wait time (1-2 minutes) and any long download steps in post
  • Credit card on file or free credits available on DO
Editing note: This video will need cuts/fast-forwards during droplet creation (~1 min), Docker image pulls, and embedding generation. Call these out when recording so you know where to cut.
Intro (0:00 - 0:30)
On Screen Narration
Show DigitalOcean dashboard, logged in. Droplets page. "Alright, let's actually set this up. We're going to create a server on DigitalOcean, install Paperless NGX and Paperless Ag, and by the end you'll have a working document archive you can search using AI."
Create the Droplet (0:30 - 3:00)
On Screen Narration
Click Click "Create" → "Droplets" in the top nav. "First we need a server. Click Create, then Droplets."
Click Region: pick the closest one (e.g., New York or San Francisco). "Pick the region closest to you. I'm going with New York but it doesn't matter much."
Click Image: select Ubuntu 24.04 (or the default LTS). Make sure it's under the OS tab, not Marketplace. "For the image, just grab Ubuntu — the latest version is fine. This is the operating system the server will run."
Click Size: scroll to the 4 GB / 2 CPU tier ($24/mo). Click it. "For size, we need at least 4 gigs of memory. This one at 24 bucks a month is the sweet spot. If you're just getting started, DigitalOcean gives new accounts 200 dollars in free credits, so you can try it out for several months at no cost."
Click Authentication: select SSH Key (recommended). If you need to add one, click "New SSH Key" and walk through it briefly. "For authentication, use an SSH key. This is like a digital key that lets you log into your server securely without typing a password every time. If you don't have one set up yet, DigitalOcean will walk you through creating one — it takes about a minute."
Click Give it a hostname like "paperless" or "my-docs." Click "Create Droplet." "Give it a name — I'll call mine 'paperless' — and hit Create Droplet."
Wait / Cut Droplet is being created. Show the progress bar briefly, then cut/fast-forward to when it's done and shows the IP address. "This takes about a minute to spin up. I'll fast forward… Alright, it's ready. And here's our IP address — that's the address of our new server."
Connect to the Server (3:00 - 4:00)
On Screen Narration
Click Copy the IP address from the DigitalOcean dashboard. Switch to your Terminal. "Now we need to connect to it. Copy that IP address and open up a terminal."
Type Type: ssh root@YOUR_IP_ADDRESS and hit Enter. Accept the fingerprint prompt if shown. "Type 'ssh root@' and paste your IP address. If it asks about a fingerprint, just type 'yes.' And we're in — we're now on our server."
Run the Install Script (4:00 - 7:00)
On Screen Narration
Type Type or paste: curl -fsSL https://paperless.fullstack.ag/install.sh | bash "Now the fun part. We just run one command. This script handles everything — it installs Docker, sets up the database, and gets Paperless and our search add-on running."
Wait The installer will show a banner and start asking questions. Walk through each prompt as it appears. "It'll ask us a few questions as it goes."
Type Answer installer prompts as they come:
- Install directory (accept default /root/paperless-ag)
- Paperless admin username
- Paperless admin password
- Domain name (if you have one, otherwise skip)
Walk through each one, explaining your choice. 		"Install directory — the default is fine. Pick a username and password for your Paperless login. If you've got a domain name you want to use, enter it here — otherwise just skip it and you can access it by IP address."
Wait / Cut Script downloads Docker images and starts services. This can take 3-5 minutes. Show a few seconds of progress, then cut/fast-forward to when it finishes. "Now it's pulling down all the pieces and starting everything up. This takes a few minutes depending on your internet speed. I'll skip ahead…"
Show The installer's final output. It should show the Paperless URL, MCP connection details, and the MCP auth token. Pause here so viewers can see it. "And we're done. The script tells us everything we need — here's the URL to access Paperless, and here are the details we'll need later to connect Claude. I'd save these somewhere."
Verify It Works (7:00 - 8:30)
On Screen Narration
Click Switch to your browser. Navigate to http://YOUR_IP:8000. The Paperless login page should appear. "Let's make sure it's working. Open your browser and go to your server's IP address, port 8000."
Type Log in with the username and password you chose during install. "Log in with the username and password you just set up. And there it is — Paperless is running. It's empty right now, but you can start uploading documents right away."
Click Optionally drag-and-drop a PDF into Paperless to show it ingesting a document. "You can just drag and drop a PDF right into the browser and Paperless will process it — read the text, add it to your archive, and our search add-on will automatically pick it up."
Connect to Claude (8:30 - 10:30)
On Screen Narration
Show Go back to the terminal where the install output is still showing the MCP connection details. "Now let's connect this to Claude so you can search your documents with plain English. Remember those connection details from the install? We need those now."
Click Open Claude Desktop. Go to Settings → Developer → Edit Config (or show the config file location). "Open up Claude Desktop and go to Settings, then Developer, then Edit Config. This opens a file where we tell Claude how to connect to our Paperless server."
Type Paste the Claude Desktop config from the install output into the config file. It uses the mcp-remote bridge. Save the file. "Paste in the config the installer gave you. This tells Claude where your server is and gives it permission to search your documents. Save the file."
Click Restart Claude Desktop. Open a new chat. Look for the MCP tools indicator (hammer icon or similar). "Restart Claude, open a new chat, and you should see the tools icon showing it's connected to Paperless."
Type Type a test query in Claude, like: "What chemicals did I spray on my corn this year?" Watch it call the MCP tool and find the Chemical Application Records. "Let's try it. 'What chemicals did I spray on my corn this year?' And there it goes — it found the chemical application records and pulled out the products, rates, everything. You're up and running."
Wrap-Up (10:30 - 11:00)
On Screen Narration
Show Claude's response with search results visible. "And that's the whole thing — from zero to a searchable document archive with AI in about ten minutes of actual work. Start scanning your paperwork and it'll all be searchable from here on out. If you've got a NAS at home, check out our other video on setting this up on a QNAP or Synology."
