Paperless Ag

Semantic search and Claude integration for Paperless NGX. Built for farmers.

Install

SSH into your Linux server and run:

curl -fsSL https://paperless.fullstack.ag/install.sh | bash

The installer will walk you through setup. It works on any VPS (DigitalOcean, Hetzner, Linode, Vultr) and detects if you already have Paperless NGX running.

What It Does

  • Adds semantic search to Paperless NGX (search by meaning, not just keywords)
  • Connects to Claude via MCP so you can search your documents through conversation
  • Runs locally with no external API calls -- your documents stay on your server
  • Paperless NGX stays completely stock.

Connect to Claude

After installation, connect your MCP server to Claude. The install script prints Claude Code and .mcp.json examples with your server URL and token. Claude Desktop uses those same values in the mcp-remote bridge config below.

Claude Code:

claude mcp add --transport http paperless-ag YOUR_SERVER_URL/mcp \
  --header "Authorization: Bearer YOUR_MCP_TOKEN"

VS Code / Cursor (.mcp.json):

{
  "mcpServers": {
    "paperless-ag": {
      "type": "http",
      "url": "YOUR_SERVER_URL/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_MCP_TOKEN"
      }
    }
  }
}

Claude Desktop (claude_desktop_config.json):

Claude Desktop doesn't support remote HTTP servers directly. Use mcp-remote as a bridge. On macOS, open Claude Desktop, choose Claude > Settings from the menu bar, go to Developer, and click Edit Config. On Windows, edit %APPDATA%\Claude\claude_desktop_config.json: 

{
  "mcpServers": {
    "paperless-ag": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "YOUR_SERVER_URL/mcp",
        "--allow-http",
        "--header",
        "Authorization:Bearer YOUR_MCP_TOKEN"
      ],
      "env": {
        "PATH": "YOUR_NODE_BIN_DIR:/usr/local/bin:/opt/homebrew/bin:/usr/bin:/bin"
      }
    }
  }
}

Replace YOUR_NODE_BIN_DIR with the directory containing your node binary. Claude Desktop doesn't inherit your shell's PATH, so it needs this to find npx and node. Find it by running: 

dirname "$(which node)"

If you use nvm or fnm, use the real path instead of the ephemeral shell path: 

dirname "$(readlink -f "$(which node)")"

If your server uses HTTPS, you can drop the --allow-http flag and the env block may not be needed if node is in a standard location like /usr/local/bin.

Save the file, quit Claude Desktop completely, and reopen it. In a new chat, open the connectors/tools menu and confirm paperless-ag is enabled.

Replace YOUR_SERVER_URL with the URL from your install output (https://yourdomain.com if you configured a domain, or http://YOUR_IP if not) and YOUR_MCP_TOKEN with the token from the install output. Lost your token? Check the .env file in your install directory (defaults to /root/paperless-ag): 

grep MCP_AUTH_TOKEN /root/paperless-ag/.env

Requirements

  • A Linux server with 4+ GB RAM (DigitalOcean $24/mo or Hetzner $10/mo recommended)
  • Docker and Docker Compose (the installer can set these up)
